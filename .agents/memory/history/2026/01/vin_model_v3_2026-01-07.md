# VINv3 (VIN-Core) implementation

## What changed
- Added `VinModelV3`/`VinModelV3Config` in `oracle_rri/oracle_rri/vin/model_v3.py` with fixed, evidence-backed defaults (field_dim=16, gn_groups=4, pool_grid=6, head 192x3, dropout 0.05, semidense proj grid=16/max_points=4096).
- Removed PointNeXt, frustum MHCA, obs-count plumbing, and voxel-valid feature concat from VIN-Core; kept optional trajectory encoder with pooled embedding (no cross-attn).
- Added `VinV3ForwardDiagnostics` in `oracle_rri/oracle_rri/vin/types.py` (leaner debug payload).
- Integrated v3 into `oracle_rri/oracle_rri/vin/__init__.py` and `oracle_rri/oracle_rri/lightning/lit_module.py` (config union + plot guard).
- Moved PointNeXt encoder to `oracle_rri/oracle_rri/vin/experimental/pointnext_encoder.py` with lazy OpenPoints import; added `oracle_rri/oracle_rri/vin/experimental/__init__.py`.
- Added effective VIN config logging to W&B in `oracle_rri/oracle_rri/lightning/lit_module.py` (writes `vin_effective.json`, logs artifact + derived divisors).
- Docs update: `docs/contents/impl/vin_nbv.qmd` now lists VINv3 and clarifies trajectory encoder behavior.

## Notes / potential follow-ups
- `oracle_rri.vin.__init__` still imports plotting (seaborn). Tests stub seaborn; consider lazy import or optional dependency guard if this becomes a recurring issue.
- Candidate validity in v3 now uses strict `> 0` checks for voxel + semidense coverage; if this is too aggressive for sparse scenes, consider adding explicit thresholds back as optional config.
- If you want all non-core encoders moved to experimental, consider migrating shell pose encoders and `spherical_encoding.py` (currently needed by `VinModelV1_SH`).
