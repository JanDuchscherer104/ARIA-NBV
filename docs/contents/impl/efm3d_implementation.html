1 EFM3D Implementation Details

The EFM3D codebase provides additional utilities for working with point clouds, meshes, and voxel grids in the context of egocentric reconstruction. This document covers the key utilities relevant for RRI computation.

1.1 Mesh Utilities

1.1.1 eval_mesh_to_mesh()

Source: efm3d/utils/mesh_utils.py

Purpose: Comprehensive mesh-to-mesh evaluation with visualizations.

Enhancements over ATEK:

  • Supports both file paths and trimesh.Trimesh objects
  • Generates visualization point clouds colored by error
  • Returns precision/recall visualizations with green (inlier) / red (outlier) coloring
  • Computes metrics at multiple thresholds (1cm, 5cm)

Output:

  • metrics: Dictionary with accuracy, completeness, precision@0.05, recall@0.05, fscore@0.05
  • viz: Dictionary with colored point clouds for visualization
  • raw_data: Per-point accuracy and completeness arrays

1.2 Point Cloud Utilities

Source: efm3d/utils/pointcloud.py

1.2.1 Core Point Cloud Operations

1.2.1.1 get_points_world()

Source: pointcloud.py#L29

Converts depth images or point cloud data to world coordinates:

  • Handles ARIA depth maps via backprojection
  • Transforms from camera → rig → world coordinate frames
  • Filters invalid depth values (NaN, zero, or negative)

1.2.1.2 collapse_pointcloud_time()

Source: pointcloud.py#L110

Merges temporal point cloud sequences:

  • Flattens time dimension
  • Removes NaN values
  • Deduplicates points using torch.unique()

1.2.1.3 pointcloud_to_voxel_ids()

Source: pointcloud.py#L121

Converts 3D points to voxel grid indices:

Algorithm:

# Compute voxel size
dW = (x_max - x_min) / vW
dH = (y_max - y_min) / vH
dD = (z_max - z_min) / vD

# Convert point to voxel index
voxel_id = floor((point - voxel_min) / [dW, dH, dD])

Validation: Ensures points lie within voxel grid bounds

1.2.1.4 pointcloud_to_occupancy_snippet()

Source: pointcloud.py#L182

Creates occupancy grid from point cloud with free-space reasoning:

Three-phase process:

  1. Camera origins: Mark as free space
  2. Ray marching: Sample points along rays from camera to surface → mark as free
  3. Surface points: Mark occupied at point locations

Parameters:

  • S: Number of samples along each ray (default: 1)
  • Uses conservative sampling: stops at depth - voxel_diagonal to avoid marking surface voxels as free

1.2.2 Occupancy Grid Construction

1.2.2.1 pointcloud_occupancy_samples()

Source: pointcloud.py#L281

Generates training samples for occupancy prediction:

Returns three point sets:

  • Occupied: Points beyond surface (at depth + voxel_diagonal)
  • Surface: Original point cloud points
  • Free space: Sampled points along rays before surface (at random depths < depth - voxel_diagonal)

Use case: Provides supervision for learning-based occupancy prediction

1.2.2.2 pointcloud_to_voxel_counts()

Source: pointcloud.py#L380

Converts point cloud to voxel grid with point density:

Output: Voxel grid where each cell contains count of points inside

  • Useful for understanding reconstruction density
  • Can be normalized and thresholded to create binary occupancy

1.3 Ray Utilities

Source: efm3d/utils/ray.py

The ray utilities provide fundamental ray-casting operations for camera modeling, coordinate transformations, and scene intersection tests. These are essential for simulating visibility, generating synthetic views, and understanding what would be observed from candidate viewpoints.

1.3.1 Ray Generation

1.3.1.1 ray_grid()

Source: ray.py#L67

High-level function to generate a grid of rays for a camera in rig coordinate system:

Purpose: Creates rays for every pixel in a camera’s image plane, useful for: - Ray-casting from candidate viewpoints - Visibility analysis - Synthetic depth rendering

Input: - cam: CameraTW object (1D or 2D batch)

Returns: - rays: Tensor of shape [H, W, 6] or [B, H, W, 6] containing [ray_origin_xyz, ray_direction_xyz] in rig coordinates - valid: Boolean mask indicating valid rays (some pixels may be outside fisheye FoV)

Example:

from efm3d.aria.camera import CameraTW
from efm3d.utils.ray import ray_grid

# Create camera
camera = CameraTW(...)

# Generate ray grid
rays, valid = ray_grid(camera)
# rays.shape: [H, W, 6] where rays[..., :3] = origins, rays[..., 3:] = directions

Key features: - Rays are in rig coordinate system (not camera coordinates) - Handles both single cameras and batched cameras - Automatically filters invalid pixels (e.g., outside fisheye projection)

1.3.1.2 grid_ray()

Source: ray.py#L24

Lower-level function that unprojects a 2D pixel grid into 3D rays:

Purpose: Given pixel coordinates, generate corresponding 3D rays

Process: 1. Unproject 2D pixels to 3D directions in camera frame (via camera.unproject()) 2. Normalize ray directions 3. Transform from camera frame to rig frame using T_camera_rig 4. Concatenate ray origins and directions into 6D representation

Returns: - rays: [B, H, W, 6] tensor - valid: [B, H, W] boolean mask

When to use: - Use ray_grid() for standard camera ray generation - Use grid_ray() when you have custom pixel coordinates or need more control

1.3.2 Ray Coordinate Transformations

1.3.2.1 transform_rays()

Source: ray.py#L80

Transforms rays from one coordinate frame to another:

Purpose: Convert rays between coordinate systems (e.g., rig → world, camera → rig)

Algorithm:

# Transform ray origins (translation + rotation)
ray_origins_new = T_new_old.transform(rays_old[..., :3])

# Transform ray directions (rotation only)
ray_directions_new = T_new_old.rotate(rays_old[..., 3:])

# Concatenate
rays_new = torch.cat([ray_origins_new, ray_directions_new], dim=-1)

Input: - rays_old: Rays in old coordinate frame [..., 6] - T_new_old: Transformation from old to new frame

Returns: - rays_new: Rays in new coordinate frame [..., 6]

Example use case:

from efm3d.utils.ray import ray_grid, transform_rays

# Generate rays in rig coordinates
rays_rig, valid = ray_grid(camera)

# Transform to world coordinates
rays_world = transform_rays(rays_rig, T_world_rig)

Critical for: - Ray-casting to GT mesh (needs rays in world coordinates) - Multi-view consistency checks - Coordinate frame alignment

1.3.3 Ray-Scene Intersection

1.3.3.1 ray_obb_intersection()

Source: ray.py#L90

Computes ray intersection with axis-aligned bounding boxes (voxel grids):

Algorithm (Slab method):

  1. Test ray against 6 planes (±X, ±Y, ±Z)
  2. For each plane:
    • Compute intersection parameter \(t\): \(t = \frac{(\mathbf{p}_{\text{plane}} - \mathbf{p}_{\text{ray}}) \cdot \mathbf{n}}{{\mathbf{d}_{\text{ray}}} \cdot \mathbf{n}}\)
    • Check if intersection point lies within plane bounds
  3. Track minimum \(t_{\text{min}}\) (entry) and maximum \(t_{\text{max}}\) (exit)
  4. Valid intersection if \(t_{\text{min}} < t_{\text{max}}\)

Input: - rays_v: Rays in voxel coordinates [B, N, 6] - voxel_extent: [x_min, x_max, y_min, y_max, z_min, z_max] - t_min, t_max: Depth range to consider - return_points: If True, return 3D intersection points

Returns: - If return_points=False: ts tensor [B, N, 2] with [t_entry, t_exit] - If return_points=True: (ts, ps_min_v, ps_max_v) with entry/exit points

Use cases: - Determine if rays intersect reconstruction volume - Compute valid depth ranges for sampling - Visibility culling

1.3.3.2 sample_depths_in_grid()

Source: ray.py#L196

Samples depths along rays within voxel grid bounds:

Purpose: Generate depth samples along rays, constrained by: 1. Voxel grid boundaries (via ray_obb_intersection()) 2. Maximum depth from sensor (e.g., GT depth or max range) 3. Minimum/maximum depth thresholds

Algorithm: 1. Compute safe voxel extent (shrunk by half-voxel to avoid boundary issues) 2. Find ray-grid intersection parameters \((t_{\text{min}}, t_{\text{max}})\) 3. Clamp depths: depths_min = max(t_min, ds_min), depths_max = min(t_max, ds_max) 4. Generate num_samples depths between depths_min and depths_max 5. Optionally add random noise

Input: - rays_v: Rays in voxel coordinates [B, T, N, 6] - ds_max: Maximum depth per ray [B, T, N] - voxel_extent: Scene bounds - W, H, D: Voxel grid dimensions - num_samples: Number of depth samples per ray - d_near, d_far: Global depth range - sample_mode: 'uniform' or 'random' - ds_min: Optional minimum depth per ray

Returns: - depths: Sampled depths [B, T, N, num_samples] - depths_max: Effective maximum depth per ray [B, T, N] - valid: Boolean mask for rays with valid intersections [B, T, N]

Use cases: - Free-space sampling for occupancy supervision - NeRF-style volumetric rendering - Visibility reasoning

Example:

# Sample 64 depths along rays within voxel grid
depths, depths_max, valid = sample_depths_in_grid(
    rays_v=rays_voxel,
    ds_max=gt_depth,
    voxel_extent=voxel_extent,
    W=128, H=128, D=128,
    num_samples=64,
    sample_mode='random'
)
# depths.shape: [B, T, N, 64]

1.3.4 Complete Ray-Casting Pipeline for RRI

Here’s how these functions work together for candidate view simulation:

from efm3d.aria.camera import CameraTW
from efm3d.utils.ray import ray_grid, transform_rays

# 1. Generate rays for candidate camera (in rig coordinates)
rays_rig, valid = ray_grid(candidate_camera)

# 2. Transform rays to world coordinates
rays_world = transform_rays(rays_rig, T_world_rig)

# 3. Intersect rays with GT mesh (using trimesh)
locations, index_ray, index_tri = gt_mesh.ray.intersects_location(
    ray_origins=rays_world[..., :3].reshape(-1, 3).cpu().numpy(),
    ray_directions=rays_world[..., 3:].reshape(-1, 3).cpu().numpy()
)

# Result: 3D points that would be visible from candidate view
P_candidate = torch.from_numpy(locations).float()

1.4 Reconstruction Utilities

Source: efm3d/utils/reconstruction.py

1.4.1 compute_occupancy_loss_subvoxel()

Source: reconstruction.py#L122

Training loss for occupancy prediction:

Strategy:

  1. Sample occupied, surface, and free-space points
  2. Query predicted occupancy via trilinear interpolation
  3. Compute loss against ground truth labels

Loss types:

  • focal: Focal loss (handles class imbalance)
  • ce: Binary cross-entropy
  • l1, l2: Regression losses
  • logl1: Log-space L1

Key feature: Subsampling support to reduce memory usage

1.5 Depth Utilities

Source: efm3d/utils/depth.py

1.5.1 dist_im_to_point_cloud_im()

Source: depth.py#L20

Converts depth images to 3D point clouds:

Algorithm:

  1. Generate ray grid for camera using ray_grid()
  2. Project rays to depth: \(\mathbf{p}_{\text{3D}} = \mathbf{o}_{\text{ray}} + d \cdot \mathbf{d}_{\text{ray}}\)
  3. Transform from rig coordinates to camera coordinates
  4. Filter invalid depths (≤ 0)

Supports: Batched operation over time and batch dimensions

1.6 Voxel Sampling Utilities

Source: efm3d/utils/voxel_sampling.py

1.6.1 sample_voxels()

Source: voxel_sampling.py#L79

Samples voxel grid features at 3D point locations using trilinear interpolation:

Implementation:

  • Normalizes points to \([-1, 1]\) range
  • Uses torch.nn.functional.grid_sample() (non-differentiable w.r.t. points)
  • Alternative: diff_grid_sample() for differentiable sampling

Use case: Query occupancy or features at arbitrary 3D locations

1.6.2 pc_to_vox()

Source: voxel_sampling.py#L20

Converts point cloud coordinates to normalized voxel coordinates:

Handles:

  • List-based or tensor-based voxel extents
  • Batched point clouds
  • Validity checking (points within bounds)

1.7 Usage for RRI Computation

The EFM3D utilities are particularly useful for the RRI oracle pipeline:

  1. Point cloud construction: Use dist_im_to_point_cloud_im() to convert depth maps to 3D points
  2. Temporal merging: Use collapse_pointcloud_time() to merge multiple views
  3. Ray-casting simulation: Use ray_obb_intersection() for visibility testing
  4. Voxel operations: Use pointcloud_to_voxel_counts() for consistent downsampling

See RRI Computation for the complete pipeline.

### Ray-Scene Intersection

#### `ray_obb_intersection()`

**Source**: [`ray.py#L90`](https://github.com/facebookresearch/efm3d/blob/main/efm3d/utils/ray.py#L90)

3. Compute loss against ground truth labels

**Loss types**:

- `focal`: Focal loss (handles class imbalance)
- `ce`: Binary cross-entropy
- `l1`, `l2`: Regression losses
- `logl1`: Log-space L1

**Key feature**: Subsampling support to reduce memory usage

## Depth Utilities

**Source**: [`efm3d/utils/depth.py`](https://github.com/facebookresearch/efm3d/blob/main/efm3d/utils/depth.py)

### `dist_im_to_point_cloud_im()`

**Source**: [`depth.py#L20`](https://github.com/facebookresearch/efm3d/blob/main/efm3d/utils/depth.py#L20)

Converts depth images to 3D point clouds:

**Algorithm**:

1. Generate ray grid for camera using `ray_grid()`
2. Project rays to depth: $\mathbf{p}_{\text{3D}} = \mathbf{o}_{\text{ray}} + d \cdot \mathbf{d}_{\text{ray}}$
3. Transform from rig coordinates to camera coordinates
4. Filter invalid depths (≤ 0)

**Supports**: Batched operation over time and batch dimensions

## Voxel Sampling Utilities

**Source**: [`efm3d/utils/voxel_sampling.py`](https://github.com/facebookresearch/efm3d/blob/main/efm3d/utils/voxel_sampling.py)

### `sample_voxels()`

**Source**: [`voxel_sampling.py#L79`](https://github.com/facebookresearch/efm3d/blob/main/efm3d/utils/voxel_sampling.py#L79)

Samples voxel grid features at 3D point locations using trilinear interpolation:

**Implementation**:

- Normalizes points to $[-1, 1]$ range
- Uses `torch.nn.functional.grid_sample()` (non-differentiable w.r.t. points)
- Alternative: `diff_grid_sample()` for differentiable sampling

**Use case**: Query occupancy or features at arbitrary 3D locations

### `pc_to_vox()`

**Source**: [`voxel_sampling.py#L20`](https://github.com/facebookresearch/efm3d/blob/main/efm3d/utils/voxel_sampling.py#L20)

Converts point cloud coordinates to normalized voxel coordinates:

**Handles**:

- List-based or tensor-based voxel extents
- Batched point clouds
- Validity checking (points within bounds)

## Usage for RRI Computation

The EFM3D utilities are particularly useful for the RRI oracle pipeline:

1. **Point cloud construction**: Use `dist_im_to_point_cloud_im()` to convert depth maps to 3D points
2. **Temporal merging**: Use `collapse_pointcloud_time()` to merge multiple views
3. **Ray-casting simulation**: Use `ray_obb_intersection()` for visibility testing
4. **Voxel operations**: Use `pointcloud_to_voxel_counts()` for consistent downsampling

See [RRI Computation](rri_computation.qmd) for the complete pipeline.
