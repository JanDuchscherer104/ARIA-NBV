## Pose-Generation Flow

SyrenkaFlowchart summary of `CandidateViewGenerator.generate` in `pose_generation/candidate_generation.py`.

```{mermaid}
%%| echo: false
---
title: CandidateViewGenerator.generate flow
---
flowchart TB
    start --> ctx
    ctx --> loop_cond
    loop_cond -->|Yes| seed
    seed --> rules
    rules --> mask
    mask --> any
    any -->|Yes| accum
    any -->|No| update
    accum --> update
    update --> loop_cond
    loop_cond -->|No| return
    start(("call generate(last_pose, gt_mesh?, occupancy_extent?)"))
    ctx["init CandidateContext (device, occupancy bounds, empty accumulators)"]
    loop_cond{"remaining > 0 and attempts < max_resamples?"}
    seed["_seed -> allocate poses[n,12], mask=ones"]
    subgraph rules["Rule sequence (order)"]
        rule_shell["ShellSamplingRule: sample centres + look-at orientations"]
        rule_mindist["MinDistanceToMeshRule: drop centres closer than min_distance"]
        rule_path["PathCollisionRule (if enabled): ray from last_pose to centre"]
        rule_free["FreeSpaceRule (if enabled): keep inside occupancy AABB"]
    end
    mask["stack rule masks -> mask_valid = all rules"]
    any{"mask_valid contains True?"}
    accum["append valid poses; collect masks + shell_poses"]
    update["remaining = num_samples - collected; attempts += 1"]
    return((("return CandidateSamplingResult (poses, mask_valid, masks, shell_poses)")))
```
