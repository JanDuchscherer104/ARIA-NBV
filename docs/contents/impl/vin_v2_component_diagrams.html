VIN v2: Component Architecture Diagrams (Mermaid)

This page contains Mermaid architecture diagrams of the major components in oracle_rri/oracle_rri/vin/model_v2.py (VIN v2).

The goal is to make it easy to cross-check the implementation against the paper and to reason about ablations (e.g., enabling/disabling semidense frustum MHCA, PointNeXt, trajectory attention, voxel-valid-fraction gating).

1 1) Pose preparation + pose encoder

Show code 
flowchart LR
    cand["Candidate poses<br/>PoseTW world←cam<br/>(B×N)"] --> cw
    ref["Reference pose<br/>PoseTW world←rig_ref<br/>(B)"] --> cw
    cw["Undo CW90 correction<br/>(optional)"] --> compose
    compose["Compose candidate pose in rig frame<br/>T rig_ref←cam = T world←rig_ref^-1 · T world←cam"] --> enc
    enc["PoseEncoder.encode()<br/>R6D + LFF"] --> pose_enc["pose_enc<br/>(B×N×d_pose)"]
    enc --> pose_vec["pose_vec<br/>(B×N×9)"]
    enc --> center["candidate_center_rig_m<br/>(B×N×3)"]

2 2) EVL scene field construction (voxel heads → compact field)

Show code 
flowchart LR
    occ_pr["EVL occ_pr"] --> field_aux
    occ_in["EVL occ_input"] --> field_aux
    cent["EVL cent_pr"] --> field_aux
    counts["EVL counts"] --> counts_norm

    counts_norm["counts_norm<br/>log1p(counts)/log1p(max)"] --> unknown
    counts_norm --> field_aux

    unknown["unknown = 1 - counts_norm"] --> new_surface
    new_surface["new_surface_prior = unknown · occ_pr"] --> field_aux

    free_in["EVL free_input<br/>(or derived)"] --> field_aux

    field_aux["Select channels<br/>scene_field_channels"] --> concat

    proj["Conv3d 1×1×1 + GroupNorm + GELU"] --> field["Field F<br/>(B×C×D×H×W)"]
